# Helm Charts

Two Helm charts deploy the LangChoice application to Kubernetes. Both are managed by ArgoCD — you never run `helm install` manually in production. ArgoCD watches the Git repo and syncs automatically when `values.yaml` changes.

## Charts

```
helm-charts/
├── langchoice-frontend/    Next.js frontend deployment
└── langchoice-backend/     Go Gin backend + MongoDB subchart
```

## langchoice-frontend

Deploys the Next.js frontend pod, service, configmap, ingress, and HPA.

### Key values

```yaml
# helm-charts/langchoice-frontend/values.yaml

image:
  repository: iamridoydey/langchoice-frontend
  tag: "abc1234"    # updated by Jenkins on every build — do not change manually
  pullPolicy: IfNotPresent

replicaCount: 1

service:
  type: ClusterIP
  port: 80
  targetPort: 3000

config:
  BACKEND_URL: "http://langchoice-backend.langchoice.svc.cluster.local:80"

ingress:
  enabled: true
  className: "alb"
  annotations:
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/load-balancer-name: langchoice-alb
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTP":80}]'
```

The `config.BACKEND_URL` value is mounted as a ConfigMap and injected into the pod as an environment variable. The Next.js route handler reads it at request time — not build time.

## langchoice-backend

Deploys the Go Gin backend and a MongoDB StatefulSet (via the Bitnami MongoDB subchart).

### Key values

```yaml
# helm-charts/langchoice-backend/values.yaml

image:
  repository: iamridoydey/langchoice-backend
  tag: "abc1234"    # updated by Jenkins

config:
  PORT: "5000"
  DB_NAME: "langchoice"
  ALLOWED_ORIGINS: "http://langchoice-alb-xxx.us-east-1.elb.amazonaws.com"
  MONGODB_URI: "mongodb://langchoice:$(MONGODB_PASSWORD)@langchoice-backend-mongodb:27017/langchoice"

mongodb:
  architecture: "standalone"
  auth:
    enabled: true
    existingSecret: mongodb-secret    # synced from AWS Secrets Manager by External Secrets
    usernames:
      - "langchoice"
    databases:
      - "langchoice"
```

The MongoDB password is never stored in Git. `existingSecret: mongodb-secret` tells the Bitnami chart to read credentials from the Kubernetes secret created by External Secrets Operator.

## How image tags are updated

Jenkins runs this `sed` command after building a new image:

```bash
sed -i 's|tag: ".*" # frontend|tag: "a3f9c12" # frontend|' helm-charts/langchoice-frontend/values.yaml
```

The `# frontend` and `# backend` comments at the end of the tag line act as unique identifiers so `sed` targets exactly the right line and nothing else.

ArgoCD detects the changed `values.yaml` within 3 minutes and triggers a rolling update.

## Testing a chart locally

```bash
# Dry run — renders templates without applying
helm template langchoice-frontend helm-charts/langchoice-frontend \
  --values helm-charts/langchoice-frontend/values.yaml

# Lint
helm lint helm-charts/langchoice-frontend

# Install manually (bypassing ArgoCD — for debugging only)
helm install langchoice-frontend helm-charts/langchoice-frontend \
  --namespace langchoice \
  --create-namespace \
  --values helm-charts/langchoice-frontend/values.yaml
```

## Templates overview

Each chart contains standard templates generated by `helm create` and then customised:

| Template | Purpose |
|---|---|
| `deployment.yaml` | Pod spec — image, env vars from ConfigMap, probes |
| `service.yaml` | ClusterIP service — internal cluster access |
| `configmap.yaml` | Non-sensitive env vars (URLs, ports, names) |
| `ingress.yaml` | ALB Ingress — only enabled for frontend |
| `hpa.yaml` | Horizontal Pod Autoscaler |
| `serviceaccount.yaml` | Service account (IRSA annotation if needed) |
| `_helpers.tpl` | Name helper functions |
